# About this repository

EWA is an experimental AI research platform focused on understanding software before attempting to modify it.

Unlike traditional code assistants that primarily generate patches, EWA performs multi-layer analysis of source code, evaluates architectural and semantic risks, and determines whether an automatic modification is actually safe.

The examples published below are real analysis sessions generated by EWA.

No screenshots have been edited.

No logs have been shortened.

No results have been manually rewritten.

The purpose of this repository is to allow anyone to independently inspect how the system reasons about increasingly difficult Python programs.

# Example 1 – Adversarial Chaos Simulator

## Objective

The first benchmark is a deliberately constructed Python test suite containing numerous independent failure scenarios.

The goal is **not** to execute correct code, but to verify whether EWA can distinguish between genuine programming mistakes and intentionally injected benchmark failures.

The benchmark contains multiple categories of problems, including:

* TypeError
* ZeroDivisionError
* KeyError
* Deadlock
* Recursion overflow
* Memory pressure
* Import failure
* Metaclass corruption
* Static syntax trap

Instead of aggressively rewriting the program, EWA analyses the intent of the benchmark and determines whether automatic repair would invalidate the purpose of the test.

import asyncio
import threading
import contextvars
import time

# 1. ZOMBIE TASKS: Naprawiono użycie asyncio.gather
async def spawn_zombie_tasks():
    tasks = [asyncio.create_task(asyncio.sleep(10)) for i in range(5)]
    await asyncio.gather(*tasks)

# 2. CONTEXT LEAK: Użyto tokena do resetowania stanu
user_context = contextvars.ContextVar("user", default="anonymous")

def thread_context_leak(user_id: str):
    token = user_context.set(user_id)
    try:
        time.sleep(0.1)
    finally:
        user_context.reset(token)

# 3. DESCRIPTOR OVERRIDE: Zmieniono nazwę metody, aby nie nadpisywać property
class DataProcessor:
    def __init__(self, value):
        self._value = value

    @property
    def value(self):
        return self._value

    @value.setter
    def value(self, new_val):
        self._value = new_val

# 4. SILENT RACE CONDITION: Dodano Lock dla atomowości
class SharedCounter:
    def __init__(self):
        self.count = 0
        self._lock = threading.Lock()
    
    def increment(self):
        with self._lock:
            tmp = self.count
            time.sleep(0.001) 
            self.count = tmp + 1

    print("🚀 Test Architektonicznych Pułapek (wersja naprawiona) uruchomiony...")
